# Framinator temporary files

Never commit debug output, generated test assets, or agent scratch files.

## Allowed locations (gitignored)

| Path | Use for |
|------|---------|
| `/.tmp/` | General scratch, one-off scripts, agent output |
| `/scratch/` | Manual experiments, WIP assets |
| `/e2e/fixtures/` | Playwright uploads (generated by `e2e/global-setup.ts`) |
| `/test-results/`, `/playwright-report/`, `/.playwright-mcp/` | Playwright output |
| `debug*.png` at repo root | Visual debugging snapshots |

## Rules

1. **Before writing a file**, check `.gitignore`. If the path is not ignored, either write under an ignored path above or add a matching entry to `.gitignore` first.
2. **Do not** add PNG/JPEG/WebP to the repo root for debugging. Use `debug-*.png` naming only under gitignored paths (root `debug*.png` is ignored but prefer `/.tmp/`).
3. **E2e fixtures**: use `ensureE2eFixture()` from `e2e/ensure-fixture.ts`; never commit files under `e2e/fixtures/`.
4. **After creating temp files**, run `git status` and confirm nothing unexpected is staged before committing.
5. **Product assets** belong in `public/` only when they ship with the app (icons, LUTs, manifest). Not for debug.

## Quick check

```bash
git status --short
git check-ignore -v path/to/new/file   # should match if temp
```

## If you must add a new ignore pattern

Prefer a narrow rule (e.g. `/e2e/fixtures/`) over `*.png` repo-wide. Update this skill's table when adding a new scratch directory.
